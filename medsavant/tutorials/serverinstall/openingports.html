Opening ports

Several ports need to be opened for the server to connect to clients, download data, and notify administrators via email.

36800 (default*) bidirectional Connections with clients
36801 (default*) bidirectional Data transfer with clients
80 incoming Data access and updates
443 outgoing Mail

* You can change the defaults by setting the 'listen-on-port' parameter in the server configuration file. See the previous section for details.

Opening Ports

Instructions for opening ports vary between operating systems. Consult reference manuals for your operating system for instructions on open ports.

The following commands may work for your distribution:

sudo iptables -A INPUT -p tcp --dport 36800 ACCEPT;
sudo iptables -A INPUT -p tcp --dport 36801 ACCEPT;
sudo iptables-save;

Testing Ports

To test if a port has been sucessfully opened, you can use netcat. On the server machine, where the ports have been opened, do

nc -l <port>

While that's waiting, on a client machine, do

nc <server_address> <port>

Then, type some words on the client machine and hit enter/return. If the port is open, the server will echo the words you typed on the client.

Once the server is running and ports are opened, you'll be able to start administrating the installation through the MedSavant client. Proceed to the Administration tutorial for instructions.
