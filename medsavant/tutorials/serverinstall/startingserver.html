Starting the MedSavant Server

The MedSavant Server manages connections from MedSavant Clients to the Infobright database.

Getting the MedSavant server

You can obtain a copy of the MedSavant server by building the source in our SVN Repository or by contacting us for the latest stable build. The latter is preferred.

Building the MedSavant Server from Source

Downloading the source

Our codebase is available on GitHub. To download it, do

git clone https://github.com/compbio-UofT/medsavant.git

Building the projects in NetBeans

In the previous step you downloaded 3 NetBeans projects: MedSavantClient, MedSavantServerEngine, and MedSavantShared. The latter two are required to build the MedSavant Server.

To build these projects you need to open them in NetBeans. If you don't already have NetBeans, we recommend installing the "NetBeans + JDK Bundle". A Google search should take you to the latest version of the bundle.

Configuring the Server Engine

The server has a number of different configurable parameters, see the example below.

Example Configuration File 
db-user = mfiume
db-password = mypassword
db-host = localhost
db-port = 5029
listen-on-port = 36800
email = myemail@address.com
tmp-dir = /tmp/ms-temporary
ms-dir = /home/ms-permanent
Starting the MedSavant Server

Once you've configured the server and database to your liking, you can start the server by doing

java -jar -Djava.rmi.server.hostname=<public_ip_of_server> MedSavantServerEngine.jar -c <path/to/config> &
