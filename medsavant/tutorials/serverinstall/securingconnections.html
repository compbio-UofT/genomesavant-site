Securing Connections using SSL

If you will be connecting to a MedSavant server outside of your private network, and the data hosted by your server is sensitive it is highly recommended to set up SSL encryption to ensure secure transfer of data.

Creating a public-private key pair

Create the Server Key

Create a self-signed server key by typing the following on the command line

keytool -genkey -v -keyalg RSA -keystore server.keystore -validity 36500 -dname "CN=Server, OU=Production, O=University of Toronto, L=Toronto, ST=Ontario, C=Canada"

The -validity argument specifies how long the key is valid for, in days.

The -dname argument specifies the distinguished name for a certificate associated with the key. The six parameters have the following significance:

  • CN - Common Name of the certificate owner
  • OU - Organizational Unit of the certificate owner
  • O - Organization to which the certificate owner belongs
  • L - Locality name of the certificate owner
  • S - State or province of the certificate owner
  • C - Country of the certificate owner

Adjust these as appropriate, but keep CN=Server

You will be prompted to enter a password. Remember it, we'll need it later.

Create the Client Key

Create a self-signed client key by typing the following on the command line

keytool -genkey -v -keyalg RSA -keystore client.keystore -validity 36500 -dname "CN=Client, OU=Production, O=University of Toronto, L=Toronto, ST=Ontario, C=Canada"

Again, edit the -dname parameters as necessary, leaving CN=Client
